## Лаба 2 — Коллекции

### Как демонстрировать

```

1. Запустил — показываешь вывод в консоли

3. Показываешь что TreeSet отсортирован и без дубликатов
4. Показываешь что Vector сохраняет порядок добавления

### Топ-10 вопросов

**1. Что такое коллекция?**
Контейнер для хранения группы объектов. Умнее массива — динамически меняет размер, есть методы поиска/сортировки/удаления.

**2. Чем Vector отличается от ArrayList?**
Vector потокобезопасен (все методы синхронизированы), ArrayList нет. Vector появился в Java 1.0, ArrayList в Java 1.2. Сегодня ArrayList быстрее и предпочтительнее.

**3. Что такое TreeSet?**
Множество (без дубликатов) с сортировкой. Хранит элементы в дереве — при добавлении автоматически встаёт на нужное место. Операции add/remove/contains работают за O(log n).

**4. Чем Set отличается от List?**
List допускает дубликаты и сохраняет порядок. Set не допускает дубликаты. TreeSet ещё и сортирует.

**5. Что такое Iterator?**
Объект для обхода коллекции. `hasNext()` — есть ли следующий элемент, `next()` — взять следующий. Позволяет безопасно удалять элементы во время обхода через `iterator.remove()`.

**6. Зачем Comparable? Что такое compareTo()?**
Comparable — интерфейс с методом `compareTo()`. Говорит как сравнивать два объекта. TreeSet использует его для сортировки. Возвращает: отрицательное (меньше), 0 (равно), положительное (больше).

**7. Что такое equals() и hashCode()?**
`equals()` — логическое равенство объектов (по значениям полей). `hashCode()` — числовой код объекта, используется в HashMap/HashSet. Правило: если equals() возвращает true, hashCode() должен быть одинаковым.

**8. Что такое autoboxing?**
Автоматическое преобразование примитива в объект: `int → Integer`. При добавлении `int` в коллекцию Java сама оборачивает его в `Integer`.

**9. Чем TreeSet отличается от HashSet?**
TreeSet сортирует (хранит в красно-чёрном дереве, O(log n)). HashSet не сортирует (хранит в хэш-таблице, O(1) для add/contains).

**10. Почему нельзя изменять коллекцию во время обхода for-each?**
for-each использует Iterator внутри. Изменение коллекции во время итерации бросает `ConcurrentModificationException`. Для безопасного удаления используй `iterator.remove()`.

### Топ-10 вопросов

**1. Что такое абстрактный класс?**
Класс с ключевым словом `abstract`. Нельзя создать объект напрямую (`new AbstractArray()` — ошибка). Может содержать абстрактные методы (без реализации) — подклассы обязаны их реализовать.

**2. Что такое абстрактный метод?**
Метод без тела: `abstract void sort();`. Подкласс обязан написать реализацию и поставить `@Override`. Это контракт — "ты ДОЛЖЕН это реализовать".

**3. Чем абстрактный класс отличается от интерфейса?**
Абстрактный класс может иметь поля и готовые методы с кодом. Интерфейс — только константы и абстрактные методы (до Java 8). Класс может наследовать только один абстрактный класс, но реализовывать много интерфейсов.

**4. Что такое интерфейс?**
Контракт поведения. Список методов которые класс обязан реализовать. `implements IArray` — "я обещаю реализовать все методы из IArray".

**5. Что такое полиморфизм?**
Один тип переменной — разные реализации. `AbstractArray arr = new InsertSort()` — переменная типа AbstractArray, но внутри объект InsertSort. При вызове `arr.sort()` выполнится код InsertSort.

**6. Что такое @Override?**
Аннотация. Говорит компилятору: "этот метод переопределяет метод из родителя". Если написал имя метода с опечаткой — компилятор сообщит об ошибке.

**7. Зачем нужно наследование?**
Избежать дублирования кода. Общий код (поля, вспомогательные методы) пишется один раз в родителе. Подклассы добавляют только своё специфическое поведение.

**8. Что такое ссылка на метод (Method Reference)?**
Короткая запись лямбды: `Math::log` вместо `x -> Math.log(x)`. Используется там где ожидается функциональный интерфейс.

**9. Почему нельзя наследовать два класса?**
Java не поддерживает множественное наследование классов — это приводит к "diamond problem" (неоднозначность какой метод вызвать). Решение — интерфейсы: можно реализовывать сколько угодно.

**10. Что такое DoubleUnaryOperator?**
Функциональный интерфейс: принимает double, возвращает double. Позволяет передавать функции как параметры. `Math::log` соответствует ему — принимает double, возвращает double.

7. **Демонстрация ошибки:** вписываешь несуществующий файл → красная надпись

### Топ-10 вопросов

**1. Что такое исключение?**
Объект-ошибка, который "летит" по стеку вызовов. Позволяет обработать ошибку в нужном месте вместо немедленного падения программы.

**2. Чем checked отличается от unchecked?**
Checked (`extends Exception`) — Java заставляет поймать или объявить `throws`. Unchecked (`extends RuntimeException`) — обработка не обязательна. IOException — checked, NullPointerException — unchecked.

**3. Зачем создавать своё исключение StationFormatException?**
Стандартные исключения описывают технические проблемы. Своё исключение добавляет смысл задачи — "неверный формат данных станции". По имени класса сразу понятно что пошло не так.

**4. Что такое try-with-resources?**
`try (ресурс = ...)` — Java автоматически вызывает `close()` после блока, и при успехе и при исключении. Гарантирует что файл всегда закроется.

**5. Зачем BufferedReader поверх FileReader?**
FileReader читает посимвольно — медленно. BufferedReader загружает блок символов в буфер, даёт метод `readLine()`. Без буфера каждый символ = обращение к диску.

**6. Что возвращает readLine() когда файл кончился?**
`null`. Поэтому цикл: `while ((line = br.readLine()) != null)`.

**7. Что такое Stream в Java?**
Конвейер обработки данных. Не хранит данные — описывает действия. `filter` → `map` → `collect`. Ленивый — обрабатывает данные только при вызове терминальной операции.

**8. Почему нельзя бросить checked exception в лямбде?**
Функциональные интерфейсы (Function) не объявляют throws. Обходим: оборачиваем в RuntimeException, пробрасываем через Stream, разворачиваем после collect.

**9. В чём разница двух способов чтения?**
BufferedReader — императивный стиль, читает весь файл построчно. Files.lines() — функциональный стиль, ленивый поток. Результат одинаковый, разница в стиле и потреблении памяти при огромных файлах.

**10. Что такое Path и Paths.get()?**
Path — объект представляющий путь к файлу. `Paths.get("data/stations.txt")` создаёт Path из строки. Предпочтительнее старого `new File("...")` — лучше API, кроссплатформенность.

5. Нажимаешь "Создать файл…" ещё раз — числа каждый раз разные (Random)
6. Показываешь что результат всегда отсортирован и без дубликатов

### Топ-10 вопросов

**1. Чем бинарный файл отличается от текстового?**
В текстовом числа хранятся как символы ("45" = два байта). В бинарном — в двоичном представлении (45 = четыре байта, `00 00 00 2D`). Бинарный быстрее, фиксированный размер, нечитаем в блокноте.

**2. Что такое DataOutputStream / DataInputStream?**
Обёртки над байтовым потоком. Добавляют writeInt(), readInt(), writeDouble() — записывают/читают примитивы Java напрямую как байты.

**3. Что делает writeInt()?**
Записывает int как ровно 4 байта в файл. readInt() делает обратное — читает 4 байта и возвращает int.

**4. Что такое EOFException и зачем он здесь?**
End Of File Exception — readInt() ждёт 4 байта, но файл закончился. Нормальный способ узнать конец бинарного файла. Ловим в catch и выходим из цикла.

**5. Зачем TreeSet а не ArrayList?**
По заданию нужна сортировка и без повторений. TreeSet даёт это автоматически при добавлении. С ArrayList пришлось бы вручную сортировать и удалять дубликаты.

**6. Как работает allDigitsOdd()?**
Берём цифры числа справа налево через `% 10` (остаток = последняя цифра). Если цифра чётная — false. `x /= 10` убирает проверенную цифру. Повторяем пока x > 0.

**7. Что значит `d & 1 == 0`?**
Побитовое И. Последний бит числа: у нечётных = 1, у чётных = 0. Быстрая проверка чётности без деления.

**8. Зачем Files.createDirectories()?**
Создаёт папку data/ если её нет. Без этого запись файла упадёт с IOException — папки не существует.

**9. Как узнать сколько чисел в бинарном файле без чтения?**
`Files.size(file) / Integer.BYTES` — размер файла в байтах делим на 4 (размер int). Проверяем что размер кратен 4, иначе файл битый.

**10. Что такое Dialog в Vaadin?**
Модальное всплывающее окно. Пользователь не может нажать что-то на основной странице пока диалог открыт. Закрывается через dlg.close().

## Лаба Чат — Сокеты

### Как демонстрировать

```

Нужно 3 терминала:

```bash

```

2. GUI клиент — открылось окно Swing, вводишь имя
3. Консольный клиент — вводишь другое имя
4. Пишешь сообщение из GUI → оно появляется у консольного клиента
5. Пишешь из консоли → появляется в GUI
6. Закрываешь GUI → в консоли сообщение "покинул чат"

### Топ-10 вопросов

**1. Что такое сокет?**
Двухсторонний канал связи между двумя программами по сети. ServerSocket ждёт подключений, Socket — сам канал. Данные передаются через InputStream/OutputStream.

**2. Как устанавливается соединение?**
Сервер: `new ServerSocket(8088)` — начинает слушать порт. `accept()` — блокируется, ждёт клиента. Клиент: `new Socket("127.0.0.1", 8088)` — подключается. accept() разблокируется.

**3. Что такое порт?**
Число от 0 до 65535. Как номер квартиры — один компьютер может запускать много программ, каждая слушает свой порт. 8088 — произвольный выбор (не зарезервированный системой).

**4. Зачем многопоточность на сервере?**
readLine() блокирует поток пока клиент не пришлёт строку. Без потоков сервер обслуживал бы клиентов по очереди — второй ждал бы пока первый пишет. Каждый клиент в своём потоке — работают параллельно.

**5. Что такое CopyOnWriteArrayList?**
Потокобезопасный список. При изменении создаёт копию массива внутри. Нужен потому что несколько потоков одновременно могут добавлять/удалять клиентов. Обычный ArrayList сломается.

**6. Что такое Runnable?**
Интерфейс с одним методом run(). Описывает задачу для потока. `new Thread(handler).start()` — запускает run() в новом потоке. Лучше чем extends Thread — можно дополнительно наследовать другой класс.

**7. Что происходит если клиент аварийно отключится?**
readLine() бросит IOException. Блок finally выполнится в любом случае — закроет сокет, удалит из clients, разошлёт уведомление всем.

**8. Зачем autoFlush в PrintWriter?**
`new PrintWriter(out, true)` — true означает что после каждого println данные немедленно отправляются. Без этого сообщения копятся в буфере и могут не дойти до получателя.

**9. Что такое EDT в Swing?**
Event Dispatch Thread — единственный поток который имеет право изменять компоненты Swing. Сетевой поток читает сообщения — изменять UI из него нельзя. SwingUtilities.invokeLater() ставит задачу в очередь EDT.

**10. Зачем клиент запускает отдельный поток для чтения?**
scanner.nextLine() блокирует главный поток. Одновременно нужно читать сообщения от сервера (тоже блокирует). Один поток не делает два блокирующих ожидания — нужен второй поток.

3. Выполнила SQL-запрос с JOIN и CASE WHEN
4. Перевела упаковки в килограммы

### Топ-10 вопросов

**1. Что такое JDBC?**
Java Database Connectivity — стандартный API для работы с базами данных. Один и тот же Java-код работает с MySQL, PostgreSQL, SQLite — нужно только поменять драйвер и строку подключения.

**2. Что такое SQLite?**
База данных в одном файле, без отдельного сервера. Идеально для учебных задач. MySQL/PostgreSQL требуют установки и запуска сервера.

**3. Три главных класса JDBC?**
Connection — соединение с БД. Statement — выполнение SQL. ResultSet — результат запроса (строки таблицы).

**4. Зачем try-with-resources для Connection?**
Connection — ресурс который нужно закрыть. try-with-resources гарантирует вызов close() даже при исключении. Незакрытые соединения — утечка ресурсов.

**5. Что такое JOIN?**
Соединение таблиц по ключу. `JOIN product p ON m.article_id = p.article_id` — объединяет строки где совпадают ключи. Результат содержит столбцы из обеих таблиц.

**6. Что делает CASE WHEN?**
Условное выражение в SQL: если условие истинно — одно значение, иначе другое. Вместе с SUM даёт "условную агрегацию" — суммируем только нужные строки.

**7. Что такое ResultSet?**
Курсор на результат запроса. Изначально стоит перед первой строкой. rs.next() переходит к следующей строке (возвращает true если есть). rs.getInt() читает значение из текущей строки.

**8. Чем PreparedStatement отличается от Statement?**
Statement — статический запрос. PreparedStatement — с параметрами (`WHERE name = ?`). Параметры передаются отдельно — защита от SQL-инъекций и лучшая производительность при повторном выполнении.

**9. Что такое SQL-инъекция?**
Если подставлять пользовательский ввод прямо в SQL-строку, злоумышленник может ввести `'; DROP TABLE product; --`. PreparedStatement предотвращает это — параметры не интерпретируются как SQL.

**10. Почему `"кг".equals(unit)` а не `unit.equals("кг")`?**
Если unit == null и написать unit.equals("кг") — NullPointerException. А `"кг".equals(null)` просто вернёт false. Стандартная защита от NPE в Java.
